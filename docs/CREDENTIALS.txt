Kivun Terminal v1.0.6 - Credentials Reference
=============================================

This file lists every credential Kivun Terminal touches and
where it lives. Read SECURITY.txt first for the "why".

1. UBUNTU USER (inside WSL2)
----------------------------
  Configured interactively on first WSL Ubuntu boot. A terminal
  prompts you for a username and password; pick whatever you like.
  Kivun Terminal does NOT ship these in config.txt anymore (the
  old USERNAME / PASSWORD defaults were removed in v1.0.6 because
  secret-scanners flagged them).

  Used for: `sudo` inside Ubuntu (apt install, etc.)
  Scope:    Local WSL instance only. Not reachable over network.

  To change PASSWORD after install:
    1. Open Kivun Terminal
    2. Run: passwd
    3. Enter old password, then new password twice

  To change USERNAME after install you must recreate the
  distribution:
    wsl --unregister Ubuntu
    wsl --install -d Ubuntu     (you will be prompted for new name)
  Then re-run the Kivun Terminal installer.

2. CLAUDE AUTHENTICATION
------------------------
  Configured in: Claude Code CLI itself, on first launch.

  Two options:
    a) Pro/Max subscription - browser OAuth flow the first time
       you run `claude`. Token stored by Claude Code inside
       ~/.config/anthropic/ in Ubuntu.

    b) Anthropic API key - paste when Claude Code prompts, or
       set ANTHROPIC_API_KEY in your ~/.bashrc inside Ubuntu.

  Kivun Terminal never reads or stores these credentials.

3. VCXSRV (optional)
--------------------
  VcXsrv has NO credentials - it's a local display server.
  When USE_VCXSRV=true, Kivun Terminal runs it with access control
  ENABLED (DisableAC=False, no -ac flag) and authorizes the
  invoking user via `xhost +si:localuser:$USER` from the WSL side.
  Other local users and LAN peers cannot connect.

  Earlier versions shipped with `-ac` / `DisableAC=True` which
  disabled access control entirely - this was fixed in v1.0.6.

  VcXsrv is no longer auto-installed by our installer. Instead,
  the VcXsrv download page opens in your browser on request and
  you install it manually. Set USE_VCXSRV=true in config.txt once
  you have it installed.

4. NETWORK / CLOUD
------------------
  Kivun Terminal does NOT:
    - Send telemetry anywhere
    - Call home
    - Upload logs, configs, or session content
    - Require a Kivun-branded account

  The only outbound connections happen via:
    - Claude Code CLI (to api.anthropic.com / claude.ai)
    - apt-get (to Ubuntu package mirrors)
    - Your browser, if you click the VcXsrv download button (the
      installer opens the SourceForge page for manual install).

QUICK RECOVERY
--------------
If you forget the Ubuntu password:
  wsl -d Ubuntu -u root -- passwd username

If Claude Code asks you to re-authenticate and browser OAuth fails:
  claude logout
  claude login
